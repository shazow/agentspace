Specifications in this directory should include:

```md
# Name of the component

One-line description of the component.

**Status**: Completed / In-Progress / Abandoned

## Goals

High-level statement for the purpose of this component.

- List of goals that describe the scope of this component.

Out of scope:

- List of possible distractions that are explicitly out of scope.

Acceptance criteria:

- [ ] List of measurable explicit properties that represent the high-level goals above.

## Progress

- [ ] Tasks that have not been completed yet 
- [x] And tasks that have been completed.

## Appendix

Any additional notes for tracking, including: diagrams, research, future work, and code samples.
```
